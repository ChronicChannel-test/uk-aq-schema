# UK AQ v0.2.0 schema and TEST migration draft

## Locked core model

The canonical v0.2.0 core model contains:

- `connectors`
- `networks`
- `stations`
- `station_initial_metadata`
- `station_matches`
- `observed_properties`
- `timeseries`
- `observations`
- `uk_aq_ingest_runs`

Connectors represent data sources. Networks are the public/logical network
labels. Each station belongs to one connector and one network. Optional
`station_matches` rows link source stations that represent the same real-world
location without merging their source observations.

`station_initial_metadata` is immutable during normal refreshes. New station
ingests must use `insert ... on conflict (station_id) do nothing`. Metadata
that needs ongoing refresh belongs on a dedicated `stations` column or another
purpose-specific table.

## Identity rules

- connectors: `connector_code`
- networks: `network_code`
- stations: `(connector_id, station_ref)`
- timeseries: `(connector_id, timeseries_ref)`
- observations: `(timeseries_id, observed_at)`
- station initial metadata: `station_id`

`service_ref` remains available on stations and timeseries but is not part of
identity. Observations have no direct connector or network column in the final
canonical model.

The first draft is additive and TEST-only:

- Creates networks, station matches, and initial metadata.
- Renames `breathelondon` to `blondon_communities`.
- Seeds `blondon_nodes`.
- Backfills station network assignment from connector defaults.
- Copies old station metadata once.
- Links timeseries directly to observed properties.
- Adds network context to ingest runs where the mapping is clear.
- Marks the known UK-AIR SOS placeholder station removed.

The canonical seed makes only GOV.UK AURN and Breathe London public initially.
OpenAQ, Sensor.Community, and LAQN remain non-public.

## Deliberately deferred

No file in this draft:

- Applies SQL to TEST or live.
- Drops `station_metadata`, `phenomena`, `station_network_memberships`, or
  `uk_aq_networks`.
- Drops legacy station/timeseries columns.
- Drops or replaces public views and RPCs.
- Introduces a generic `connector_checkpoints` table.
- Adds a station public-display flag.
- Adds network identity to timeseries or observations.
- Changes AQI helper tables or functions.

The current observations table is partitioned by `connector_id`, and its
primary key includes that partition key. PostgreSQL cannot reach the final
v0.2.0 `(timeseries_id, observed_at)` key through a purely additive alteration.
The TEST migration therefore preserves the current table and all observation
rows, adds `metadata`, and defers the table replacement until dependencies and
copy/swap validation are designed.

## Required dependency analysis

Before destructive TEST cleanup or any live-beta migration, inspect:

- Public views and all PostgREST RPCs.
- Ingest station, timeseries, observation, and metadata upserts.
- AQI helper functions and hourly tables.
- Latest snapshot and R2 history builders.
- Website network, station, current-value, and history queries.
- Connector-specific checkpoint ownership for Breathe London Communities and
  Breathe London Nodes.

No Supabase egress change is expected from creating or backfilling these schema
objects alone. Database size will increase modestly from new metadata, indexes,
and network columns; the additive phase deliberately retains legacy columns and
tables, so it does not reclaim storage.
