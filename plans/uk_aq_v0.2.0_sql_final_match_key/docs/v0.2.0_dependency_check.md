## 8. Recommended fix order

1. Complete the additive phase and save `005b`, `009` and `010` report output.
2. Update timeseries writers and lookups to populate observed_property_id while retaining phenomenon_id compatibility.
3. Update observation RPCs and every direct writer to the new conflict key behind a coordinated deployment plan.
4. Replace public views/RPC payloads and update website consumers.
5. Update AQI, latest snapshot, history integrity, backup and repair tooling.
6. Design and test observation-table RLS, policy, grants, ownership and rollback SQL.
7. Pause writes, rehearse the hard cut on a disposable TEST copy, then review whether to enable migration 900.
8. Enforce observed_property_id NOT NULL only after `005b` returns zero exceptions.
